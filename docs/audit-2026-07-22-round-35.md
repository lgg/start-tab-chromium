# Deep audit round 35 — complete project, screen, and persistence review

## Scope

Independent review of the exact `master` state produced by PR #111. The pass covered all tracked source and configuration files, every user-facing surface (New Tab, layout editor, Options, popup, blocked page, Split View/disabled overlays), service-worker events, settings/runtime mutations, import/export and Browser Sync canonicalization, localization catalogs, manifests/build profiles, release documentation, manual QA, and the full historical CI contract.

## Confirmed defects

1. Options created and persisted a new block before its settings dialog was confirmed. Cancelling the dialog therefore left an unwanted block behind.
2. Saving that delayed block was based on a stale pre-dialog settings snapshot and grid position, allowing concurrent settings changes to be overwritten or a newly confirmed block to overlap another block.
3. Multiple block, layout, and theme helper mutations read settings outside the write lock and later replaced the whole settings object. Two Options/New Tab contexts could silently overwrite each other's changes.
4. Split View filtered only two Chromium schemes. `file:`, `javascript:`, browser-specific internal schemes, and other non-web targets could still appear and be assigned to the tab.
5. Split View and disabled overlays were visually modal but did not make the underlying Start Tab inert, expose dialog semantics, or move keyboard focus into the overlay.
6. Semantic settings/runtime/task comparisons used raw `JSON.stringify`, so identical objects with different key insertion order could trigger migrations, optimistic-concurrency failures, or unnecessary writes.
7. The Round 34 runtime echo comparator sorted only outer instance IDs; reordered keys inside clocks and task objects could still rebuild the Start Tab.
8. Browser Sync checksum canonicalization and bounded backup recovery used locale-sensitive collation. Ordering could differ across operating systems or ICU versions and invalidate otherwise identical cross-device snapshots.
9. Default block titles were handled inconsistently. The Options dialog could show an internal key such as `blockTitleNote`; Start Tab palette additions froze the current translation instead of retaining a locale-adaptive key; layout-editor accessibility/confirmation text, note labels, and clock notifications could expose internal keys. Legitimate custom titles beginning with `blockTitle` were also mistaken for internal keys.
10. Popup's destructive “clear all blocked sites” action had no confirmation.
11. Popup language selection could remain visually changed after persistence failed.
12. The blocked-page countdown left Cancel enabled while the unblock operation was already being committed, creating a misleading cancellation race at zero seconds.
13. Locale parity automation inspected only a hard-coded subset of the JSON catalogs even though runtime localization loads split catalogs. Missing or duplicate keys in an omitted catalog could pass CI.
14. Options navigation exposed an English-only accessibility label instead of the active locale.
15. README installation/navigation instructions described sections and controls that no longer matched the actual Options screen.
16. Historical Round 34 static assertions were tied to the superseded implementation shape and had to be updated to preserve the behavioral contract while accepting the stronger canonical comparator.

## Corrections

- Split block creation into a pure draft and an atomic confirm/save operation. Cancellation now makes no storage change; confirmation revalidates capacity/singletons and recalculates placement against the latest locked settings snapshot.
- Converted all block/layout/theme helper writes to `updateStartPageSettings`, so each mutation is applied atomically to the current state rather than replacing concurrent changes with stale snapshots.
- Added one recursive, locale-independent JSON canonicalizer and used it for settings normalization, runtime normalization, task conflicts, Options no-op detection, and nested runtime storage-echo comparison.
- Replaced locale-sensitive checksum/recovery ordering with deterministic code-unit ordering in Browser Sync, backup URL recovery, and rule canonicalization.
- Centralized exact default-title detection. All screens and notifications now localize only the exact descriptor key, preserve locale-adaptive defaults, and leave custom titles untouched.
- Restricted Split View candidates to parsed HTTP(S) URLs, normalized titles, and added inert/modal/focus behavior.
- Added confirmation and failure rollback in popup flows, and disabled countdown cancellation once unblock commitment begins.
- Made locale parity enumerate every JSON catalog, report duplicate keys, and localized the Options navigation label.
- Corrected README and expanded manual QA for cancellation, Split View safety, and the affected screens.
- Added executable Round 35 fixtures, static contracts, CI wiring, and updated the compatible Round 34 contract.

## Validation boundary

Repository automation can validate source contracts, migrations, fixtures, type safety, package profiles, and deterministic builds. It cannot prove physical focus behavior in every Chromium fork, real browser Split View/native-new-tab behavior, live third-party API availability, production Google OAuth, Chrome Web Store review, or cross-device interaction quality. Those remain explicit manual release checks rather than being represented as automated proof.
