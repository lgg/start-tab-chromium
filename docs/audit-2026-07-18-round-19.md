## Material findings fixed

### User-controlled identifiers collided with JavaScript object prototypes

Block instance IDs, domain names, and clock completion tokens are persisted as dictionary keys. Several runtime/settings/statistics paths used ordinary `{}` objects and direct indexed reads. Values such as `__proto__`, `constructor`, `toString`, or `hasOwnProperty` could therefore be lost, read as inherited values, or interact with the legacy `__proto__` setter. In focus-stat updates, a special domain could cause writes against `Object.prototype` instead of a domain-stat record.

A shared dictionary helper now creates null-prototype records and performs own-property-only reads. It is applied to:

- per-instance clocks, notes, tasks, and link-page positions;
- per-domain estimated-time settings;
- per-day/per-domain focus statistics and processed completion tokens;
- last-blocked URL storage;
- service-worker, settings, and new-tab consumers that read user-controlled instance IDs.

### Strict backup capacity checks also blocked local recovery

Round 18 correctly rejected oversized externally supplied backups before mutation. The same strict normalization path was also used for exporting current state and creating the pre-import recovery snapshot. If local storage was already corrupted or created by an older buggy version above supported limits, both export and import of a valid repair backup could fail precisely when recovery was needed.

Backup normalization now distinguishes:

- `strict-import`: untrusted external backups remain rejected above supported capacities before any Chrome API access;
- `local-recovery`: already-local data is deterministically normalized and bounded for export or pre-import recovery.

This does not silently accept or truncate an incoming oversized backup. It restores the ability to export and apply a valid backup while retaining a bounded recovery snapshot of the previous local state.

## Regression coverage

Round 19 executable fixtures verify:

- special block IDs survive settings/runtime normalization for notes, tasks, clocks, and link paging;
- user-keyed dictionaries have a null prototype;
- domain-minute settings, focus-domain statistics, completion tokens, and last-blocked URLs preserve special names;
- recording a `__proto__` domain does not mutate `Object.prototype`;
- export succeeds from local state above block, task, and blocked-site limits and produces supported bounded data;
- import of a valid backup succeeds from that corrupted state and retains a bounded pre-import recovery backup;
- an externally supplied blocked-site collection above the limit still fails before storage, DNR, revision, or alarm access.

Static guards verify the dictionary helper and all high-risk consumers. Generated-output checks require prototype-free dictionary creation and own-property reads in Options, service-worker, and new-tab bundles. The Round 18 guard was updated to assert strict rejection specifically on the external-import path.
