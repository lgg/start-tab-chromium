# Ten-Pass Audit Report - 2026-07-04

Repository: `lgg/start-tab-chromium`
Branch: `codex/10-pass-audit-20260704`

Each pass reviewed the whole project surface, not a single isolated component: architecture and responsibility split, new tab runtime, options UI, layout editor, site blocking, timers/stopwatch/pomodoro, IP/weather/calendar/search/history/pinned blocks, storage/local state/sync/backup/import/export, EN/RU i18n, MV3 manifest permissions, CI/build/typecheck, performance, error handling, security, UX, and preservation of existing user settings.

## Pass 1

Checked the full runtime and UI surface end to end. Found that `newtab.html` used `aria-live="polite"` on the whole layout grid while date/time and clocks can update every second. This could create noisy accessibility announcements and unnecessary live-region churn.

Fixed: removed the broad live region from the grid in `src/newtab/newtab.html`.

## Pass 2

Rechecked service worker routing, blocked-page flow, popup actions, DNR rule sync, focus stats, and timer interactions. Found that repeated clicks on the blocked page unblock button could start more than one countdown interval before the first flow completed.

Fixed: hardened `src/blocked/blocked.ts` with countdown active state, button disabling, interval cleanup, and `pagehide` cleanup.

## Pass 3

Rechecked manual backup, import/export, browser sync backup, Google Drive backup, storage keys, and local state preservation. Found that browser sync upload attempted to write arbitrarily large backup payloads. Large image/background data can exceed `chrome.storage.sync` quota and fail late with a poor UX.

Fixed: added a conservative browser sync chunk limit in `src/lib/chrome-sync.ts` and return a clear error directing users to JSON export or Google Drive backup for large backups.

## Pass 4

Rechecked fetch and observer behavior for IP, weather, Google Calendar, history, browser pinned tabs, Start Tab pinned links, and layout/instance scripts. Found that the IP helper observed the whole document even though it only needs to react to IP block insertion.

Fixed: narrowed the IP helper `MutationObserver` scope in `src/newtab/ip.js` to the grid/body fallback instead of the whole document element.

## Pass 5

Rechecked options UI, tabbed settings, JSON editors, layout editor, helper scripts, appearance presets, and preservation of settings after saves. No additional mandatory code changes were found after the earlier fixes. The helper-layer architecture remains pragmatic for the current project size, but longer-term consolidation is noted below.

Fixed: no code change required in this pass.

## Pass 6

## Remaining Risks / Technical Debt

- Some plain JavaScript helper layers still contain English fallback strings and prompt labels. They should be moved into EN/RU localization catalogs in a dedicated pass.
- `options-helper.js`, `background-presets.js`, and `instances.js` extend the TypeScript options/newtab core from separate plain JS files. This is workable, but a future consolidation into typed modules would reduce drift risk.
- Browser sync has a conservative size guard. Very large local data should use manual JSON export or Google Drive backup.
- Google Calendar and Google Drive depend on replacing the OAuth placeholder in `src/manifest.json` with a real Chrome extension OAuth client ID.
