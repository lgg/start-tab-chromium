## Material findings fixed

### Browser Sync chunks could exceed Chrome's real per-item quota

The old chunker limited raw UTF-8 text to 7000 bytes. Chrome measures each sync item using the JSON-stringified value plus the key length. Quotes, backslashes, line breaks, and other escaped characters can therefore occupy substantially more storage than their raw text size, causing a valid local backup to fail during `chrome.storage.sync.set()`.

The chunker now:

- reads the runtime `QUOTA_BYTES_PER_ITEM` value with a documented fallback;
- counts the sync key and exact JSON string escaping;
- preserves Unicode code points;
- rejects an impossible quota before attempting a remote write;
- keeps the existing whole-file and content checksums unchanged.

### A storage refresh could be lost behind a queued resize render

The new-tab page used one boolean for both visual renders and storage-backed refreshes. If resize queued a plain render first, a subsequent settings/runtime storage event was discarded and the tab could remain stale until another event occurred.

A dedicated serialized render scheduler now coalesces frames while allowing a later refresh to upgrade an already queued render. Refreshes run before rendering, async operations cannot overtake each other, errors are reported, and disposed pages stop scheduling work.

### Stale Options statistics could append after a newer render

Statistics and migration status render asynchronously. Overlapping Options renders could allow an older promise to append a duplicate or stale section after the current page was already rebuilt.

Every Options render now carries a generation token. Only the latest generation may append its asynchronous statistics section or report its failure.

### UI actions could create unhandled promise rejections

Onboarding preset actions, settings opening, layout save/configure actions, and several Options edit/create dialogs launched promises without a visible rejection path. They now use shared UI error reporters; the Layout Editor exposes an explicit `onError` channel.

## Regression coverage

Round 12 fixtures verify:

- quote/backslash/newline-heavy and Unicode-heavy sync chunks stay within the exact JSON-stringified per-item quota;
- chunks reconstruct the exact original backup JSON;
- impossible quotas fail early;
- a queued render upgraded by a refresh performs refresh then render in one frame;
- a later visual render cannot downgrade a queued refresh;
- disposed render schedulers do not schedule new work.

Static guards prevent restoration of raw-byte chunking, the shared render boolean race, stale Options async appends, and unhandled Layout Editor actions.

## Validation boundary

Repository automation can validate source, migrations, deterministic mocks, failure injection, strict TypeScript, generated packages, and dependency advisories. It cannot prove Chrome Web Store review, production Google OAuth consent, live Calendar/Drive responses, true multi-device Sync propagation and quota throttling, every Chromium-derived browser's native-new-tab behavior, or physical pointer/touch/high-DPI/screen-reader interaction. Those remain explicit interactive checks rather than falsely claimed automated results.
