# Deep audit round 33 — current-state feature truthfulness and UI boundaries

## Scope

Independent full-project review of the exact `master` state produced by PR #109. The pass compared README and release claims with the manifest, build profiles, Options and Start Tab UI, block settings, runtime persistence, migration/backup paths, Browser Sync, browser-derived data, Google/weather integrations, and the complete regression chain.

## Confirmed defects

1. Note persistence enforced a 200,000-character limit in several backend boundaries, but the textarea exposed no matching limit. Users could type text that appeared accepted and was then silently discarded after saving or reload.
2. A configured weather city that failed geocoding silently fell back to stored coordinates. The result could therefore show weather for a different location while the block still presented the configured city.
3. Recent-history and browser-pinned blocks accepted any URL returned by browser APIs. Internal or privileged schemes could become clickable even though user-managed links consistently enforce HTTP(S).
4. Switching the Options language from an explicit override back to Auto removed the override but did not reload, so the effective locale stayed stale until a manual page refresh.
5. Editing, removing, or renaming search providers did not update the default-provider select. The dialog could display and submit a provider ID that no longer existed.
6. README called the Options UI tabbed even though the production page uses persistent section navigation.

## Corrections

- Introduced one shared `MAX_NOTE_LENGTH` used by recovery, message validation, service-worker persistence, and the textarea `maxLength` boundary.
- Made coordinates an explicit weather mode used only when the city field is blank; configured-city lookup failures now surface the existing unavailable state.
- Routed browser history and pinned-tab entries through the shared HTTP(S) URL validator.
- Reloaded Options after every locale preference change, including Auto.
- Added live collection notifications and rebuilt the default-provider select while provider rows are edited.
- Corrected README terminology.
- Added executable Round 33 boundary fixtures, static contracts, and full CI wiring.

## External boundary

Repository automation cannot prove live Open-Meteo availability, real Chrome history contents, browser-specific internal-page behavior, production Google OAuth/API behavior, Chrome Web Store review, or physical interaction quality. These remain manual release checks and are not represented as automated proof.
