## Material findings fixed

### Reset-all clocks could partially commit

The service worker implemented **Reset all clocks** by enumerating clock IDs and performing one complete runtime/alarm transaction per instance. A failure while resetting a later instance left earlier clocks already reset. Another runtime action could also enter the queue between individual resets, so the user-facing command had no single atomic linearization point.

The command now executes as one serialized runtime job and one `mutateStartPageRuntimeStateWithAlarms` transaction. It resets every currently configured Timer, Stopwatch, and Pomodoro together, preserves unrelated note/task/link runtime, reconciles the complete alarm set once, and restores exact prior runtime/revision storage and alarms after any storage or alarm failure.

Interrupted active Pomodoro work sessions are returned by that committed transaction and recorded with one batch focus-statistics mutation, preserving the correct interruption count and total focus duration.

### Failed native-new-tab fallback left an orphan tab

The browser-native new-tab escape first creates an active `about:blank` tab and then tries browser-specific internal URLs. When every URL was rejected or bypass consumption timed out, the bypass marker was removed but the temporary tab remained open.

Native-tab ownership now lives in a dedicated testable module. Failed attempts remove only their own bypass marker and close the exact temporary tab. Cleanup failures are preserved together with the primary navigation failures in an `AggregateError`; successful native tabs remain open and stop the fallback sequence immediately.

### Blocklist capacity exceeded Chrome's redirect-rule quota

The extension accepted replacement payloads containing up to 10,000 blocked sites and generated one dynamic redirect rule per site. Chrome classifies redirect rules outside the safe-rule set and limits unsafe dynamic rules to 5,000. A blocklist above that boundary therefore could not be represented by the declared implementation.

A centralized `MAX_BLOCKED_SITES = 5_000` limit now applies before any persistent or DNR change in:

- service-worker message validation;
- incremental block additions;
- complete blocklist replacement;
- legacy blocklist migration;
- DNR rule construction;
- backup export normalization and backup/import migration.

Oversized data is rejected explicitly rather than silently truncated, so the stored blocklist, DNR rules, and backup semantics cannot diverge.

## Regression coverage

Round 16 executable fixtures verify:

- a forced partial alarm-clear rejection restores exact runtime/revision storage and the complete prior alarm set;
- successful reset-all resets all clock types together and preserves unrelated runtime;
- active Pomodoro work interruptions are returned with exact elapsed durations;
- batch focus-statistics recording counts and sums all valid interrupted sessions atomically;
- rejected native-new-tab URLs remove the owned bypass marker and close the temporary tab;
- successful bypass consumption keeps the native tab open and stops fallback attempts;
- exactly 5,000 blocked sites are accepted by capacity and message validation;
- 5,001 sites are rejected;
- an oversized backup fails during migration before storage or DNR mutation.

Static and generated-output guards prevent regression to per-instance reset loops, unowned native-tab creation, missing temporary-tab cleanup, or an oversized blocklist message boundary.
